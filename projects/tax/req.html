Project: Tax Calculator App

Create a Tax Calculator that calculates tax and total price from a user’s input. Follow the mockup layout and the requirements below. Use HTML + CSS + JavaScript (no frameworks).

Design Mockup (SVG)

Match this layout: Header + centered card with inputs, buttons, and result panel.

1. Required Files

  • index.html (tax calculator UI)
  • style.css (all styling)
  • script.js (calculator logic)

2. Required Inputs

  • Price (subtotal) input (number)
  • Tax rate input (number, percent)

Example: price = 1000, tax rate = 18 → tax = 180, total = 1180

3. Required Buttons

  • Calculate button
  • Reset button

4. Required Output (Results)

  • Show calculated Tax amount
  • Show calculated Total price (subtotal + tax)
  • Show results clearly in a result panel area

5. Calculation Rules

  • Tax = price * (taxRate / 100)
  • Total = price + tax
  • Round the results to 2 decimal places

6. Validation Rules (Important)

  • If price is empty or not a number → show error message
  • If price is negative → show error message
  • If tax rate is empty or not a number → show error message
  • If tax rate is negative → show error message
  • After error, user must be able to fix input and calculate again

7. Reset Rules

  • Reset clears inputs and results
  • Reset removes error messages
  • After reset, default result display should be 0 or blank

8. Bonus Features (Optional Extra Points)

  • Add a dropdown for common tax rates: 0%, 5%, 10%, 18%
  • Support currency symbol display (e.g. RWF, USD, JPY)
  • Calculate automatically while typing (no button) — optional
  • Save last used tax rate in localStorage

9. Design & Technical Rules

  • Follow the mockup layout and spacing
  • Make it responsive (mobile friendly)
  • Use semantic HTML where possible
  • No frameworks (Bootstrap/Tailwind)
  • No inline styles — all CSS in style.css

10. Submission Checklist

  • Inputs work and accept valid numbers
  • Calculate shows correct tax and total
  • Validation errors show correctly
  • Reset works correctly
  • Layout matches mockup
  • Code is clean and readable

Tip: Build HTML first → style the card → then implement JS logic with validation.
