Portfolio Website Project

Your task: build a one-page portfolio that matches the design image (mockup) below. Copy the layout, spacing, and section order as closely as possible.

Design Image (SVG Mockup)

Match this mockup: Navbar → Hero → About + Favorites (two columns) → Contact.

1. Required Files

  • index.html (main page)
  • style.css (all styling here; no inline styles)
  • assets/ folder (profile photo, icons if used)

2. Required Sections

  1. Header / Navbar (name + links)
  2. Hero (big title + short intro + button)
  3. About Me (3–5 sentences)
  4. Favorites (Food + Hobbies lists)
  5. Contact (email + phone/social + button)
  6. Footer (copyright)

3. Content Rules

  • Use real information (name, short bio, favorites).
  • Favorites must include 2 lists (Food and Hobbies) with at least 3 items each.
  • Contact must include at least one: email, phone, or social link.

4. Design Rules (Important)

  • Follow the SVG mockup layout: spacing, sections, two-column area.
  • Make it responsive: works on mobile and desktop.
  • Use CSS for layout (recommended: flex or grid).
  • Readable text (good contrast, not too small).

5. Technical Rules

  • Use semantic tags: <header>, <main>, <section>, <footer>.
  • Navigation links must use anchors with section IDs (example: href="#about").
  • No frameworks (no Bootstrap/Tailwind).
  • JavaScript is optional (menu toggle only).

6. Submission Checklist

  • All required sections exist and are in the correct order.
  • Navbar links scroll correctly to sections.
  • Two-column layout for About + Favorites (like the mockup).
  • CSS is in style.css and linked properly.
  • No broken images/links.

Tip: Build HTML structure first, then apply CSS to match the mockup.
