Introduction to CSS

CSS (Cascading Style Sheets) is the language used to control the design of HTML: layout, colors, fonts, spacing, positioning, and responsive behavior. HTML is the structure; CSS is the appearance.

CSS is not only “beauty”. It is also for readability, usability, accessibility, and responsive layout.

1) Definition, Utility, Types

✅ Definition

CSS is a style language that selects HTML elements and applies visual rules. “Cascading” means rules can override each other depending on priority.

Example: all h1 are blue, but one special h1 can be red using a class.

✅ Utility (Why CSS?)

  • Improve readability (font size, line height, spacing)
  • Create layout (columns, navbars, grids, card UI)
  • Branding (colors, style, consistency)
  • Responsive design (mobile / tablet / desktop)
  • Better UX (hover effects, focus styles, transitions)

✅ Types (Broad meaning)

Inline CSS Internal CSS External CSS Imported CSS

These describe where CSS is written and how the browser loads it. You’ll learn each one below.

2) Creating CSS files + CSS syntax

✅ Creating CSS files

  1. Create a file named style.css
  2. Link it in HTML using <link>
<!-- index.html -->
<head>
  <link rel="stylesheet" href="style.css">
</head>

The browser requests style.css and applies rules to the HTML.

✅ CSS Syntax

A CSS rule has: selector + declaration block.

selector {
  property: value;
  property: value;
}
PartMeaningExample
Selector Which element(s) to style h1, .card, #header
Declaration One property + value color: red;
Declaration Block Many declarations inside { } { color:red; font-size:20px; }

Always end each declaration with ;

Selector Examples (Most common)

/* Element selector */
p { color: #111827; }

/* Class selector */
.card { border: 1px solid #e5e7eb; }

/* ID selector */
#main-title { font-size: 32px; }

/* Group selector */
h1, h2, h3 { font-weight: 700; }

/* Descendant selector */
.nav a { text-decoration: none; }

/* Pseudo-class */
button:hover { transform: translateY(-1px); }

/* Attribute selector */
input[type="email"] { border: 1px solid #93c5fd; }
Priority rule (simple): inline > id > class > element. Same priority → the last rule wins (cascade).

3) Use of CSS style types

✅ Inline

CSS written directly on an HTML element.

<h1 style="color: red; font-size: 30px;">
  Hello
</h1>

Bad for large projects. Hard to maintain.

✅ Internal

CSS inside <style> in the HTML file.

<head>
  <style>
    body { font-family: Arial; }
    h1 { color: blue; }
  </style>
</head>

Okay for small practice files.

✅ External

CSS in a separate file. Best for real websites.

<link rel="stylesheet" href="style.css">

Good for reusability and organization.

✅ Imported

Load CSS inside another CSS file using @import.

/* style.css */
@import url("layout.css");
@import url("colors.css");

body { background: white; }

It works, but external <link> is usually faster/cleaner.

4) CSS Visual rules (Font, Text, Color, Background, Opacity, Images)

Fonts

body{
  font-family: Arial, sans-serif;
  font-size: 16px;
  font-weight: 400; /* normal */
}

h1{
  font-size: 32px;
  font-weight: 700; /* bold */
}

Text rules

p{
  line-height: 1.8;
  letter-spacing: 0.2px;
  word-spacing: 1px;
  text-align: left; /* center / right / justify */
}

a{
  text-decoration: none;
}

Color + Background

.btn{
  color: white;
  background-color: #2563eb;
  border: 1px solid #1d4ed8;
}

Opacity

.disabled{
  opacity: 0.4; /* 0 = invisible, 1 = solid */
  pointer-events: none;
}

Background image

.hero{
  background-image: url("banner.jpg");
  background-size: cover;      /* contain / cover */
  background-position: center; /* top / bottom / left / right */
  background-repeat: no-repeat;
}
Use color for text, background-color for backgrounds.

5) CSS Display and positioning

✅ Display types

/* block: takes full width, new line */
div{ display: block; }

/* inline: only content width, cannot set width easily */
span{ display: inline; }

/* inline-block: inline but can set width/height */
.badge{ display: inline-block; width: 120px; }

/* none: removed from layout */
.hidden{ display: none; }

✅ Positioning

.box{
  position: relative; /* base reference for children */
}

.box .child{
  position: absolute;
  top: 10px;
  right: 10px;
}

.sticky-nav{
  position: sticky;
  top: 0;
}

.floating{
  position: fixed;
  bottom: 14px;
  right: 14px;
}
  • relative: keeps space, moves visually
  • absolute: removed from normal flow
  • fixed: stays on screen on scroll
  • sticky: becomes fixed after reaching top

Example: Badge on top-right of a card

<div class="product">
  <span class="badge">NEW</span>
  <h3>Shoes</h3>
  <p>Nice running shoes</p>
</div>
.product{
  position: relative;
  border: 1px solid #e5e7eb;
  padding: 16px;
  border-radius: 12px;
}

.badge{
  position: absolute;
  top: 12px;
  right: 12px;
  background: #2563eb;
  color: white;
  padding: 4px 10px;
  border-radius: 999px;
  font-size: 12px;
}

6) CSS Box model

Every element is a “box”. Box model = content + padding + border + margin. 

.card{
  width: 320px;       /* content width */
  height: auto;

  padding: 16px;      /* space inside border */
  border: 1px solid #e5e7eb;
  border-radius: 12px;

  margin: 20px 0;     /* space outside box */
}

Border + radius

.pill{
  border: 2px solid #93c5fd;
  border-radius: 999px; /* makes it pill */
  padding: 6px 12px;
}

Visibility vs display

.invisible{
  visibility: hidden; /* still takes space */
}

.removed{
  display: none; /* removed from layout */
}

Auto (centering block)

.container{
  width: 900px;
  margin: 0 auto; /* center horizontally */
}
Best practice: use * { box-sizing: border-box; } so width includes padding/border.

7) Applying Typography on webpage

✅ Introduction to typography

  • Typography means how text looks and feels.
  • Goal: easy to read + consistent style.
  • Important: font size, line height, spacing, contrast.

✅ CSS scripts for typography (examples)

body{
  font-family: system-ui, Arial, sans-serif;
  font-size: 16px;
  line-height: 1.8;
  color: #111827;
}

h1{
  font-size: 34px;
  letter-spacing: .2px;
}

p{
  max-width: 70ch; /* comfortable reading width */
}

✅ Use HTML tags required for typography

<h1>Main Title</h1>
<h2>Section Title</h2>
<p>Paragraph text...</p>

<strong>Bold important text</strong>
<em>Italic emphasis</em>
<small>Small note text</small>
<blockquote>Quoted text</blockquote>

Embed and position project object in webpage content

“Project object” can mean an embedded image, video, map, iframe, or a section showing a project. 

<div class="project">
  <img src="project.png" alt="Project screenshot">
  <div class="project-text">
    <h3>My Portfolio</h3>
    <p>A simple website project...</p>
  </div>
</div>
.project{
  display: flex;
  gap: 14px;
  align-items: center;
  border: 1px solid #e5e7eb;
  padding: 14px;
  border-radius: 14px;
}

.project img{
  width: 180px;
  border-radius: 12px;
}

8) Setting media query rules (Responsive Design)

✅ Definition

Media queries apply CSS only when conditions are true (screen size, orientation, etc). This is how we support desktop, tablet, and smartphone.

✅ Identify device types (common)

  • Desktop: usually above 1024px width
  • Tablet: around 768px - 1024px
  • Smartphone: usually below 768px

✅ Breakpoints example

/* default = desktop */
.grid-layout{
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 14px;
}

/* tablet */
@media (max-width: 1024px){
  .grid-layout{
    grid-template-columns: repeat(2, 1fr);
  }
}

/* smartphone */
@media (max-width: 768px){
  .grid-layout{
    grid-template-columns: 1fr;
  }
}

✅ Media orientation

@media (orientation: portrait){
  body{ font-size: 16px; }
}

@media (orientation: landscape){
  body{ font-size: 17px; }
}

✅ Hide elements with media queries

.desktop-only{ display: block; }
.mobile-only{ display: none; }

@media (max-width: 768px){
  .desktop-only{ display: none; }
  .mobile-only{ display: block; }
}

✅ Variable Font Size (Responsive typography)

h1{
  font-size: 34px;
}

@media (max-width: 768px){
  h1{
    font-size: 26px;
  }
}
Responsive design rule: start with a clean layout, then adjust for smaller screens.
