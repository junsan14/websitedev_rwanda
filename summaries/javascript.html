Introduction to JavaScript

JavaScript is the language that makes websites interactive. HTML builds the structure, CSS controls the design, and JavaScript adds behavior (logic + interaction). In this document, every topic includes a short explanation and practical examples you can copy.

Rule: If the website needs “click / type / change / validate / animate”, you use JavaScript.

1) Definition, application, and key concepts

Definition of JavaScript

JavaScript is a programming language mainly used in web browsers. It can read and update HTML/CSS, respond to user actions (click, input), and communicate with servers. 

// Example: simple JS running in browser
console.log("Hello JavaScript");

Application of JavaScript

JavaScript is used for user interaction and dynamic updates. Common use cases:

  • Form validation (check required fields)
  • DOM update (change text, create elements)
  • UI behavior (tabs, modal, dropdown)
  • Animation (class toggle + CSS transition)
  • Saving data (localStorage)
// Example: change text after click
// (DOM example is later in the DOM section)

JavaScript key concepts (with explanation)

These are words you must know to read and write JavaScript code.

Variable Data Type Value Operator Expression Keyword Comment

Variable

A variable is a “box” that stores a value. We use it so we can reuse data later. 

let name = "Alice"; // store a string value
let age = 20;       // store a number value

Data Types

Data type is the kind of value stored (number, string, boolean, object...). Types matter because different types behave differently (ex: "2" + 2 becomes "22"). 

let n = 10;         // number
let s = "10";       // string
let b = true;       // boolean

Operators

Operators do actions: add, compare, assign. Example: +, ===, =. 

let total = 10 + 5;
let ok = total === 15;

Expressions

An expression is code that produces a value. Example: 2+3 produces 5. 

2 + 3           // -> 5
age >= 18       // -> true/false
"Hi " + name    // -> string

Keywords and comments

Keywords are reserved words used by the language. Comments are notes for humans.

let, const, if, for, function, return

2) Integration of JavaScript to HTML

Why integration matters

HTML alone cannot run logic. JavaScript must be loaded using a <script> tag. Where you place the script matters because the browser reads HTML from top to bottom.

JavaScript inside HTML (head vs body)

In <head> (can run too early)

<head>
  <script>
    console.log("head script");
  </script>
</head>
If you try to access HTML elements here, they may not exist yet.

Before </body> (recommended)

<body>
  <h1 id="title">Hello</h1>

  <script>
    console.log("body-end script");
  </script>
</body>
Best for beginners: script at the end so the HTML loads first.

External JavaScript file

Using an external file keeps code clean. You write JS in app.js and link it. 

<script src="app.js"></script>

CDN (external library link)

A CDN loads a library from the internet. This is optional for your course. 

<script src="https://cdn.jsdelivr.net/npm/axios/dist/axios.min.js"></script>

3) Variables

What variables do

Variables store data, and you can use that data in calculations, conditions, and DOM updates. Use let for changeable values, and const for fixed references. 

let score = 10;   // can change
score = 20;

const school = "RTB";  // should not change

Naming conventions

Use meaningful names. Use camelCase for variables. 

let studentName = "Alice";
let totalScore = 90;
const MAX_USERS = 100;

Re-declaration rule

let and const cannot be redeclared in the same scope. This prevents confusing bugs. 

let x = 1;
// let x = 2; // ❌ error

var y = 1;
var y = 2; // ✅ allowed (but not recommended)

4) Data types + type casting

Why types matter

JavaScript is dynamically typed, meaning you don’t declare a type, but the value has a type. Type mistakes cause common bugs like string-number confusion.

Primitive vs reference types

// Primitive
let n = 10;         // number
let s = "10";       // string
let b = true;       // boolean
let u;              // undefined
let z = null;       // null

// Reference
let arr = [1,2,3];            // array
let obj = {name:"A"};         // object
let fn = () => "hello";       // function

Type casting (convert types)

When you get user input from HTML, it is usually a string. Convert to number if needed. 

let inputValue = "20";      // string from input
let num = Number(inputValue); // 20 (number)

let a = "12.9";
parseInt(a);   // 12
parseFloat(a); // 12.9
Example bug: "2" + 2 becomes "22". Use Number() when you want math.

5) Operators

Why operators are important

Operators allow math, comparisons, and logic. They are used everywhere in conditions and loops.

Arithmetic + assignment

let x = 10;
x += 5; // 15
x -= 3; // 12
x *= 2; // 24
x /= 4; // 6
x %= 4; // 2

Comparison (use ===)

Use === for strict comparison (type + value). == can cause unexpected conversions. 

5 == "5"   // true (loose)
5 === "5"  // false (strict)

Logical operators

let hasAccount = true;
let hasPassword = false;

hasAccount && hasPassword // false
hasAccount || hasPassword // true
!hasAccount               // false

Ternary operator

A short if-else. Good for simple conditions. 

let age = 17;
let type = age >= 18 ? "Adult" : "Child";

6) Strings

Why strings matter

Most website data is text: names, emails, messages. You will often format and validate strings.

Template literals (recommended)

let name = "Alice";
let msg = `Hello ${name}, welcome!`;

Common string methods

let t = "  JavaScript  ";

t.trim();          // remove spaces
t.toUpperCase();   // uppercase
t.includes("Java");// search
t.replace("Java","Type");

7) Condition statements

Why conditions are used

Conditions make decisions. Example: if login is correct → go to dashboard, else show error. 

let score = 75;

if(score >= 80){
  console.log("A");
}else if(score >= 60){
  console.log("B");
}else{
  console.log("C");
}

Switch (good for fixed options)

let day = "Mon";

switch(day){
  case "Mon":
    console.log("Start");
    break;
  case "Fri":
    console.log("Finish");
    break;
  default:
    console.log("Normal");
}

8) Loops

Why loops are used

Loops repeat code. Example: show many students from an array, or calculate total marks.

For loop (most common)

for(let i = 1; i <= 5; i++){
  console.log(i);
}

For/of (arrays)

const nums = [10, 20, 30];

for(const n of nums){
  console.log(n);
}

While loop

let x = 3;
while(x > 0){
  console.log(x);
  x--;
}

9) Functions

Why functions exist

Functions organize code into reusable blocks. Instead of repeating logic, you call a function.

Function definition + call

function add(a, b){
  return a + b;
}

let result = add(3, 4); // 7

Arrow function

const multiply = (a, b) => a * b;

Closure (simple explanation)

A closure happens when a function remembers variables from its outer function. This can create “private” state like a counter. 

function makeCounter(){
  let count = 0;
  return function(){
    count++;
    return count;
  }
}

const counter = makeCounter();
counter(); // 1
counter(); // 2

10) Objects

Why objects are used

Objects group related data and functions. Example: a student has name, age, class, and methods. 

const student = {
  name: "Alice",
  age: 18,
  className: "S6",
  sayHi(){
    return "Hi " + this.name;
  }
};

student.name;      // "Alice"
student.sayHi();   // "Hi Alice"

Map and Set

Use Set for unique values, and Map for key-value storage (like object but more flexible). 

const set = new Set([1,1,2]); // {1,2}
set.add(3);

const map = new Map();
map.set("name", "Jun");
map.set("age", 30);

11) Arrays

Why arrays are important

Arrays store lists of values. Websites often show lists: students, posts, products, messages. 

const students = ["Alice", "Bob", "Chris"];
students.push("Diana");

Array iteration (modern)

const nums = [1,2,3,4];

const doubled = nums.map(n => n * 2);
const even = nums.filter(n => n % 2 === 0);
const total = nums.reduce((sum, n) => sum + n, 0);

12) DOM (Document Object Model)

What is DOM?

The DOM is the browser’s representation of your HTML as objects. JavaScript can select elements and change them.

Select elements

document.getElementById("title");
document.querySelector(".box");
document.querySelectorAll(".item");

Update text/content

const el = document.getElementById("result");

el.textContent = "Safe text";
el.innerHTML = "<b>Bold text</b>";
Use innerHTML carefully. Never insert user input directly (security risk).

13) Events

What is an event?

Events happen when users interact: click, input, submit, keypress. JavaScript can “listen” and run code. 

<button id="btn">Click</button>
<p id="msg"></p>

<script>
  const btn = document.getElementById("btn");
  const msg = document.getElementById("msg");

  btn.addEventListener("click", () => {
    msg.textContent = "Button clicked!";
  });
</script>

14) JS + HTML styles

Two ways to change style

(1) Change inline style directly, (2) Toggle a CSS class (recommended for clean code).

Inline style

const box = document.getElementById("box");
box.style.backgroundColor = "lightblue";
box.style.padding = "20px";

Class toggle (recommended)

// CSS:
.active { transform: translateY(-2px); transition: .2s; }

// JS:
box.classList.toggle("active");

15) Window object

What is window?

The window object represents the browser window. It provides methods like alert and timers, and storage like localStorage. 

alert("Hello");
setTimeout(() => console.log("after 1s"), 1000);

localStorage.setItem("name", "Jun");
localStorage.getItem("name");

16) Form validation

Why validate?

Validation prevents bad input. Frontend validation improves user experience, but backend validation is still required for security. 

<form id="signup">
  <input id="email" placeholder="Email">
  <input id="pass" type="password" placeholder="Password">
  <button>Register</button>
  <p id="err"></p>
</form>

<script>
  const form = document.getElementById("signup");
  const err = document.getElementById("err");

  form.addEventListener("submit", (e) => {
    e.preventDefault();

    const email = document.getElementById("email").value.trim();
    const pass = document.getElementById("pass").value;

    if(email === "" || pass === ""){
      err.textContent = "All fields are required.";
      err.style.color = "red";
      return;
    }

    if(!email.includes("@")){
      err.textContent = "Email must include @";
      err.style.color = "red";
      return;
    }

    if(pass.length < 6){
      err.textContent = "Password must be at least 6 characters.";
      err.style.color = "red";
      return;
    }

    err.textContent = "OK!";
    err.style.color = "green";
  });
</script>

17) Async (Promise + async/await)

Why async exists

Some operations take time: fetching data, waiting for a timer, reading a file. Async lets your page stay responsive while waiting. 

// Promise
const p = new Promise((resolve) => {
  setTimeout(() => resolve("done"), 1000);
});

p.then(msg => console.log(msg));

// async/await
async function run(){
  const msg = await p;
  console.log("await:", msg);
}
run();

18) Regular expression (RegEx)

Why RegEx is used

RegEx is used to match patterns in strings (emails, passwords, search, validation). 

const email = "test@example.com";
const ok = /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email);

19) Error handling

Why error handling matters

Errors will happen: wrong input, missing variable, network failure. Try/catch helps your app not crash and allows you to show messages. 

try{
  JSON.parse("{bad json}");
}catch(err){
  console.log("Error:", err.message);
}

Throw your own error

function checkAge(age){
  if(age < 0){
    throw new Error("Age cannot be negative");
  }
}
try{
  checkAge(-1);
}catch(e){
  console.log(e.message);
}
